# 模型使用规则（省 token 策略）

目标：显著降低 **OpenAI codex（gpt-5.2）** 的 token 消耗，同时保持整体完成质量。

---

## 任务分层与模型分配

### L1：低价值/重复劳动（优先 MiniMax）
特点：搬运、整理、提取、格式化、翻译、轻总结。

推荐：
- **minimax（小M）** 优先承担 L1（性价比高）
- 能用脚本/工具就先不用模型（更省）

典型任务：
- 文章/网页内容转 Markdown（先用 Jina Reader）
- 从 JSON/表格抽取字段、整理成表
- 生成邮件模板初稿
- 把零散想法归类到 SKOS 目录

### L2：中等难度（MiniMax / 小gemini）
特点：需要语言理解与一定推理，但不需要高强度工程。

推荐：
- **minimax** 或 **小gemini（Gemini 2.5 Flash）**

典型任务：
- 候选筛选、对比、整理证据
- 笔记归纳、提炼要点
- 假设/决策树的第一版草稿

### L3：高难度/高风险（只用 codex）
特点：需要强工程能力、严格正确性、复杂排障。

推荐：
- **codex（gpt-5.2）**

典型任务：
- 修改 OpenClaw 配置/排障（JSON、systemd、服务异常）
- 写/改脚本、解析器、自动化流程
- 复杂系统设计（多组件联动）

---

## 一条可执行的省钱规则

> **只要不是写代码/排障/改配置，就先用 minimax；minimax 搞不定再用小gemini；最后才用 codex。**

---

## 配合工作流（文件化产出）进一步省 token

长任务尽量文件化：
- brief.md：目标/约束/交付物（不频繁变化）
- working.md：候选清单/证据/状态（频繁变化）

对话中只交流：
- 增量更新
- 关键摘要
- 文件路径
