# 🧠 Stanley Knowledge Operating System（SKOS v1.0）

> 这不是笔记系统。这是：**个人认知操作系统 | 决策驱动型知识实验室 | 可进化的认知资本引擎**

---

## 一、整体目录结构

```
00_Inbox        # 临时捕获区
10_Projects     # 有明确截止时间的目标
20_Areas       # 长期责任区
30_Resources   # 原始材料存放区
40_Archive     # 完成或失效项目归档区
50_Atomic      # 原子笔记区
60_Structures  # 跨领域结构模型
61_Models      # 整合模型区
70_Output      # 输出层（文章、决策等）
80_Hypotheses  # 假设库
81_Decision_Trees  # 决策树
82_Postmortem  # 复盘库
90_Index       # 结构与模型索引
91_Dashboard   # 系统监控面板
```

---

## 二、执行层（PARA 结构）

### 00_Inbox
- **临时捕获区**，不停留超过 7 天
- 只进不留，防止信息堵塞大脑

### 10_Projects
- 有明确截止时间的目标
- 示例：优化投资模型 / 完成 AI 工具评估
- **作用**：行动驱动

### 20_Areas
- 长期责任区
- 示例：投资 / 技术研究 / 健康 / 认知升级
- **作用**：防止知识失焦

### 30_Resources
- 原始材料存放区
- 书摘 / 论文 / 市场数据 / 文章
- 注意：只是素材仓库，不是思考区

### 40_Archive
- 完成或失效项目归档区

---

## 三、思维核心层（认知发动机）

### 50_Atomic（原子笔记区）
每条笔记只表达**一个完整判断**。

### 60_Structures（结构库）
跨领域结构模型：
- 正反馈 / 负反馈 / 临界点
- 信息不对称 / 网络效应
- 激励错配 / 非线性增长 / 博弈结构

### 61_Models（模型区）
整合 Atomic 笔记形成系统模型：
- 一级模型 / 子模型 / 决策框架

---

## 四、输出层

### 70_Output
- 文章草稿 / 决策框架 / PPT / 投资判断总结
- **原则**：没有输出的知识是未成熟的知识

---

## 五、决策驱动层

### 80_Hypotheses
假设库，记录每个假设及其触发/失效条件

### 81_Decision_Trees
决策树，避免模糊决策

### 82_Postmortem
复盘库，认知进化核心区

---

## 六、控制层

### 90_Index
结构与模型索引系统

### 91_Dashboard
- 原子笔记数量
- 模型整合数
- 假设准确率
- 决策准确率

---

## 七、运行逻辑

```
资源 → Atomic → Structures → Models
              ↓
          Hypothesis
              ↓
       Decision Tree
              ↓
         Postmortem
              ↓
          模型升级
```
