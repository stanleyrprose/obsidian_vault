# 一次登录 + 持久化 profile，把认证变成可复用资产

## 定义
通过浏览器自动化保存 provider 的登录态（profile），实现“只登录一次，后续 CLI 可复用”。

## 本质
把不可自动化的登录/验证码步骤，转化为一次性的人类动作，然后长期复用。

## 解决的问题
- API key 不可用/受限时，仍能使用网页端能力
- 需要跨模型交叉验证，但不想重复登录

## 边界
- 依赖有界面浏览器完成首次登录
- 登录态可能过期，需要重新登录刷新

## 迁移可能
适用于任何“登录一次后可持久化会话”的网页工具（不只 LLM）。
